# 墨祟：走阴录 — 战斗试玩 技术文档 v2.2

## 一、项目概况

一个纯前端的俯视角动作肉鸽原型，用 HTML5 Canvas 2D 渲染，没有框架依赖。

单局流程：选武器 → 波次战斗（9波，含关卡调制器）→ 波次间选遗物/进化 → 打Boss → 结算。

所有文件放在同一目录，双击 `game.html` 即可运行。


## 二、文件清单

| 文件 | 职责 |
|------|------|
| `game.html` | 游戏入口。武器选择、Canvas、HUD、遗物弹窗、暂停、结算的 DOM 结构 |
| `game.css` | 游戏专用样式。HUD、武器/遗物卡片、遮罩层、波次动画 |
| `styles.css` | 全站公用样式。配色变量、字体、纸张质感背景 |
| `gamedata.js` | 共享数据文件。武器/遗物/进化/波次/敌人/前置条件/封顶值等全部常量，game.html 和 wiki.html 共同引用 |
| `game.js` | 主游戏逻辑。IIFE 包裹，严格模式，约 2250 行 |
| `sound.js` | Web Audio API 合成音效。无音频文件，全部程序生成 |
| `mobile-controls.js` | 移动端虚拟摇杆。触摸检测+双摇杆+闪避/暂停按钮+横屏锁定，canvas 绘制 |
| `app.js` | 首页 `index.html` 的构筑预览逻辑（与游戏无关） |
| `wiki.html` | 百科页面。自动读取 gamedata.js 渲染武器/遗物/进化/敌人/波次/构筑线/封顶值 |
| `index.html` | 概念展示首页，含导航至试玩和百科 |


## 三、核心架构（game.js）

整体是一个立即执行函数，内部用闭包维护游戏状态。

```
gamedata.js (共享数据，game.html/wiki.html 均加载)
├── 常量：画布尺寸(W/H/A)、调色板(C)
├── 数据表：WEAPONS(4)、RELICS(40)、EVOLUTIONS(16)、WAVES(9)、ETYPE(10)、STAGE_MODS(6)
├── 规则：PREREQS(10件前置条件)、CAPS(6项封顶值)、LIMITS(8类对象上限)、RANGES(9项距离阈值)、WAVE_SCALE(2项波次缩放)
├── 映射：DEATH_COLOR、PCOL、BUILD_PREFS、JUDGMENTS

game.js (IIFE，约1591行)
├── 工具函数：dstSq（主用）/dst（仅需要实际距离时）、ang、cl、rn/ri、pick、shuf、moveScale、pushLimited
├── 性能：perfPressure、perfMul、markPerf
├── 对象入口：addFire、addEProj、pushAttack、spawnJudgment、mkMinion
├── 判词系统：spawnJudgment()
├── 游戏状态工厂：mkPlayer()、newGame()
├── 核心循环：
│   ├── pAtk()    玩家攻击（读stats，生成attack对象，含暴击/蓄力/骨血墨惩罚）
│   ├── hitE()    命中敌人（遗物触发、击杀结算，单次遍历恐惧/魂链/爆裂）
│   ├── hurtP()   玩家受伤（减免、收阴袋护盾、反伤、复活）
│   ├── update()  每帧逻辑（移动、敌人AI、碰撞、计时器、性能采样）
│   └── render()  每帧绘制（分层渲染17层）
├── 遗物选择：RELIC_RULES表驱动 + scoreRelicChoice()、pickRelicChoices()
├── UI 函数：updateHUD、showRelic、showEnd
└── 初始化：事件监听、主循环(requestAnimationFrame)
```

游戏主循环每帧执行 `update() → render()`，状态机控制流程：

```
playing → (清怪) → waveClear → (选遗物/进化) → playing → ... → victory/over
                  ↘ (ESC) → paused → (ESC) → playing
```

游戏循环使用独立 try-catch 保护 update/render/showEnd，防止单帧错误导致循环死亡。render 开头强制 `c.restore();c.save();c.translate(...)` 重置 Canvas 状态，防止偏移积累。


## 四、属性体系（stats）

玩家有一个 `stats` 对象，存放所有可被遗物/强化修改的数值属性：

```javascript
stats: {
  dmg: 1,        // 伤害倍率。武器基础伤害 × 此值
  spd: 1,        // 移速倍率。moveScale() 返回此值+击杀加速buff
  range: 1,      // 范围倍率。影响近战弧半径、弹道射程、AOE半径
  critRate: 0.2, // 暴击率。每次攻击 Math.random() < 此值 即暴击
  critDmg: 1.5,  // 暴击伤害倍率。暴击时 伤害 × 此值
  atkSpd: 1,     // 攻速倍率。冷却帧数 × 此值，越小越快
  multi: 1,      // 多重打击。1=单发，2=双发，3=三发……
  projSize: 1,   // 弹道体积倍率。影响碰撞半径和绘制大小
  def: 0,        // 伤害减免。受伤时 伤害 × (1 - 此值)
  returnInk: 0   // 折返墨迹强度。影响折返弹伤害和自动生成
}
```

**设计原则**：数值遗物改 `stats`，机制遗物改 boolean 标记。两者独立叠加，互不干扰。

**新系统：诱饵（decoys）**

留影伞骨触发时在闪避起点生成诱饵，存入 `g.decoys`。诱饵持续50帧，120范围内吸引敌人转向。每帧递减，渲染为半透明朱砂圆。

**新系统：处决判词（floatTexts）**

高价值击杀（暴击/弱点/Boss/三连段）触发判词浮字，存入 `g.floatTexts`。每帧上浮并淡出，用楷体+朱砂色渲染。判词库 8 句，随机选取。

**性能护栏：对象数量上限**

后期构筑会叠出大量 ring、火场、粒子和敌弹。为避免镇魂铃/魂链/火场组合后期卡顿，运行时先限制对象数量；范围数值上限后续统一设计，不在本轮硬切。

| 对象 | 上限 |
|------|------|
| particles | 260 |
| fires | 72 |
| attacks | 90 |
| eProj | 90 |

`floatTexts / decoys / kites / frosts` 也有轻量上限，避免后期反馈对象累积。对象进入统一走 `pushLimited/addFire/addEProj/pushAttack`，不要直接 `g.xxx.push()`。

**性能观测（v1.4.1）**

按 `T` 可查看：

| 指标 | 含义 |
|------|------|
| fps | 平滑帧率估计 |
| en / atk / prj | 敌人、玩家攻击、敌方弹数 |
| par / fire / txt / dec | 粒子、火场、判词、诱饵数 |
| kite / frs | 纸鸢、冰冻区数量 |
| peak * | 本局历史峰值 |
| pressure | 当前对象压力百分比 |

`pressure` 由 `particles / fires / attacks / eProj` 的占用率取最大值。粒子生成会按压力自动降载，火场与回血 tick 会错峰执行，避免镇魂铃/魂链/火场叠加时同帧尖峰过高。

## 十、遗物选择算法

### 三路加权选择

`pickRelicChoices(g)` 从未拥有遗物池中选出3件，分三次调用：

| 路数 | 模式 | 倾向 |
|------|------|------|
| 第一路 | main | 武器共鸣、流派补强、遗物联动 |
| 第二路 | support | 补短板（生存、控场、暴击、击杀） |
| 第三路 | wild | 变招（诅咒、火、分裂、处决、闪避） |

每次对全池打分→取分最高的前8件→加权随机→标记已选。

### 评分规则 `scoreRelicChoice()`

基础分 18 + 少量随机。加分项：

| 维度 | 条件 | 加分 | 理由 |
|------|------|------|------|
| 武器共鸣 | 标签匹配武器标签 | +10/标签 | 直接适配 |
| 流派补强 | 标签匹配 BUILD_PREFS | +7/标签 | 方向正确 |
| 遗物联动 | 标签已拥有 | min(10, count*4) | 组合收益 |
| 标签过载 | 同标签已拥有4件+ | -4 | 避免偏科 |
| 血线保护 | HP<55% + 防御标签 | +9 | 活命优先 |
| 低血风险 | HP<35% + 诅咒 | -10 | 别找死 |
| 前期稳定 | wave≤2 + 防御标签 | +4 | 开局求稳 |
| 后期清场 | wave≥5 + 输出标签 | +6 | 收割节奏 |
| 遗物专属 | 30+条id→情境规则 | 5~10 | 见下表 |

## 十一、扩展指南

### 加新遗物

在 RELICS 数组末尾追加。数值遗物改 stats，机制遗物设 boolean 标记。机制遗物还需在对应触发函数中加检查逻辑。

### 加权重

每件新遗物必须补 `scoreRelicChoice()` 中的专属规则，否则只会靠通用标签得分，适配度低。

### 加新武器

在 WEAPONS 追加，在 pAtk 的 `w.type` 分支加攻击生成逻辑。同时更新 BUILD_PREFS。

### 加新敌人/波次/调制器

ETYPE 定义新类型，WAVES 引用。新调制器在 STAGE_MODS 加名称，在 startStage/updateStage/renderStage 实装。
