### A. 文档口径统一

| 优先级 | 事项 | 交付物 |

### B. 运行时结构收口

| 优先级 | 事项 | 说明 |
|---|---|---|
| P0 | 对象入池统一 | 攻击、火场、敌弹、粒子都必须走统一入口，禁止直接 `push` |
| P0 | 命中 source 统一 | 近战击杀、魂伤、反伤、召唤物、弹体都必须带来源，避免遗物误触发 |
| P0 | 选择逻辑收口 | 遗物 / 进化 / 誓印的前置、后续项、跳过逻辑必须有统一过滤顺序 |
| P1 | 波次生命周期收口 | `startWave / cleanupWave / waveClear` 明确哪些对象清、哪些对象保留 |
| P1 | Boss 生命周期收口 | Boss 阶段、召唤物、分身、结算触发只走单一入口 |
| P1 | 状态字段登记 | 玩家运行时字段、敌人临时字段、全局计时器建立清单，避免隐式字段扩散 |

### C. 回归测试补强

把历史高发 bug 转成固定测试，不靠人工记忆。

| 类型 | 必测场景 |
|---|---|
| 遗物选择 | 不能给未解锁后续项；跳过后不残留；同波不重复弹出 |
| 进化选择 | 未选前置进化时不能出现后续进化 |
| 弹体命中 | 穿透弹同一目标不重复逐帧结算；折返弹必须初始化命中表 |
| source 判定 | 近战遗物只吃近战击杀，魂/火/反伤不能误触发 |
| 诱饵逻辑 | 敌人移动目标和攻击目标一致 |
| 每日种子 | 同日期同选择可复现，不污染普通模式 RNG |
| 长跑稳定 | 49 秒、3 分钟、无尽 10 波无 console error |

### A. 软拆分优先

不要一上来强拆 `game.js`。先做低风险软拆分：

|---|---|---|

| 5 | 再考虑运行时拆分 | 只有边界稳定后才拆 `game-core / game-render / game-ui` |

### B. 模块边界建议

未来可以演进为：

```text
gamedata.js          内容数据
game.js              启动、主循环、状态机
game-combat.js       攻击、命中、伤害、击杀
game-wave.js         波次、敌人生成、Boss 生命周期
game-render.js       Canvas 绘制
game-ui.js           DOM 弹窗、HUD、结算
game-debug.js        debug HUD、性能统计
mobile-controls.js   移动端输入
```

拆分原则：

- 每次只拆一个边界。
- 拆前加测试，拆后行为不变。
- 不在拆分同时新增玩法。
- 保持脚本加载顺序明确。
- 不引入框架，不改成构建依赖项目。

第 6 武器 / 第 5 Boss / 新大系统必须满足：

### D. v14.0 阶段 3-5 软拆分未执行说明

v14.0 路线书原计划"软拆分阶段 3-5：UI 辅助外移 / 调试外移 / 运行时拆 game-core / game-render / game-ui"。**实际未拆分**，原因：

- **游戏主结构是单 IIFE**：所有 fn 互相闭包引用（hitE → hurtP → onEnemyKilled → damageEnemy → CAPS / TUNING / LIMITS 全局引用）。拆出独立文件必须用 IIFE 嵌套暴露，会大幅改写 fn 名和调用点。
- **CLAUDE.md 锁定"不碰 hitE/hurtP/spawnWave 结构"**：拆 fn 等于改名/改调用点，违反结构锁定。
- **ROI 不划算**：game.js 5323 行 / 上限 8000 行，余量 2677 行（51% 余量），硬拆的维护收益低于风险。
- **本轮用更稳路径代替**：阶段 8 抽提 BOSSTYPES / BOSS_LABELS 集中表、阶段 9 审计 P2 数值 cap，达成"减少散落特判"目标。

**待 v15 评估**：若届时 game.js > 6500 行或团队扩张，再考虑运行时拆 game-combat.js / game-render.js。

#### B. 资料生成方式

- 优先从 `gamedata.js` 自动生成（用脚本读 .js 对象 → 渲染 markdown）

- source 规则表 + pool 规则表 + 数值公式表是 **手写**（设计意图，代码不会写出来）

### 任务分发模板

```md
目标：<版本号 + 一句话目标>

范围：
- 允许修改：
- 禁止修改：

必须遵守：
- 保持 IIFE / var / function / 双引号风格。
- 不直接 push 到 g.attacks / g.fires / g.eProj。
- 敌人扣血统一走 damageEnemy(g,e,dmg,src)。
- 新机制必须带 source 判定。
- 新遗物必须同步 PREREQS / BUILD_PREFS / RELIC_RULES / 图标 / 测试。
- 不要大改 hitE / hurtP / spawnWave。

验收：
- node --check gamedata.js
- node --check game.js
- node smoke_test.js
- node content_test.js
- npm run www

交付：
- 变更清单
- 自测结果
- 未解决问题
- 文档更新位置
```

### 审核重点

1. 有没有新增隐式字段但没默认值。
2. 有没有直接 `push` 到运行时对象池。
3. 有没有 source 判定不清导致遗物误触发。
4. 有没有选择池前置过滤漏掉后续项。
5. 有没有 UI 弹窗改变状态机。
6. 有没有热路径 `Math.sqrt()`、无限粒子、无限浮字。
7. 有没有只更新文案不更新 wiki / 测试 / 图标。
